1 Prérequis

Ce dont vous avez besoin avant de commencer :

  • Un VPS avec Ubuntu 22.04 LTS ou Debian 12 (recommandé)
  • Au minimum : 1 vCPU, 2 Go RAM, 20 Go de stockage
  • Un accès SSH root ou un utilisateur avec les droits sudo
  • Le code source du projet sur GitHub : https://github.com/Ralphlandry/Site_socoprocaap.git
  • (Optionnel) Un nom de domaine pointant vers l'IP du VPS
Fournisseurs VPS recommendés : OVH, Hostinger, DigitalOcean, Hetzner, Contabo. Comptez entre 4 € et 10 €/mois pour une configuration suffisante.

2 Connexion au VPS et configuration initiale

2.1 — Se connecter en SSH

Depuis votre ordinateur (Windows : utilisez PowerShell ou PuTTY) :

ssh root@ADRESSE_IP_DU_VPS

Si vous avez une clé SSH configurée :

ssh -i chemin\vers\votre_cle.pem root@ADRESSE_IP_DU_VPS

2.2 — Mettre à jour le système

apt update && apt upgrade -y

2.3 — Créer un utilisateur non-root (bonne pratique)

adduser socopro
usermod -aG sudo socopro
su - socopro

2.4 — Configurer le pare-feu

sudo ufw allow OpenSSH
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable
sudo ufw status
Ces règles autorisent SSH (port 22), HTTP (port 80) et HTTPS (port 443).

3 Installation de Docker et Docker Compose

3.1 — Installer Docker

sudo apt install -y ca-certificates curl gnupg lsb-release

sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

3.2 — Permettre à votre utilisateur d'utiliser Docker sans sudo

sudo usermod -aG docker $USER
newgrp docker

3.3 — Vérifier l'installation

docker --version
docker compose version
Résultat attendu :
Docker version 26.x.x
Docker Compose version v2.x.x

4 Cloner le projet sur le serveur

4.1 — Installer Git (si absent)

sudo apt install -y git

4.2 — Cloner le dépôt

cd /opt
sudo git clone https://github.com/Ralphlandry/Site_socoprocaap.git socopro
sudo chown -R $USER:$USER /opt/socopro
cd /opt/socopro
Le projet est maintenant dans /opt/socopro. Vous pouvez choisir un autre chemin si vous préférez (ex: /home/socopro/site).

5 Configurer les variables d'environnement

5.1 — Créer le fichier .env

À la racine du projet, créez un fichier .env :

nano /opt/socopro/.env

Collez le contenu suivant en remplaçant les valeurs :

POSTGRES_DB=socopro_db
POSTGRES_USER=socopro_user
POSTGRES_PASSWORD=MOT_DE_PASSE_FORT_ICI

DJANGO_SECRET_KEY=UNE_CLE_SECRETE_TRES_LONGUE_ET_ALEATOIRE
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=VOTRE_DOMAINE.COM,ADRESSE_IP_DU_VPS

VITE_API_URL=https://VOTRE_DOMAINE.COM/api
Important : Ne jamais utiliser des mots de passe simples en production. Générez une clé secrète Django avec la commande :
python3 -c "import secrets; print(secrets.token_urlsafe(50))"

5.2 — Vérifier que docker-compose.yml utilise les variables .env

Ouvrez docker-compose.yml et assurez-vous que les variables sont bien référencées :

cat /opt/socopro/docker-compose.yml

6 Lancer l'application avec Docker

6.1 — Construire et démarrer tous les services

cd /opt/socopro
docker compose up --build -d

Cette commande construit les images et démarre :

  • Le conteneur PostgreSQL (base de données)
  • Le conteneur backend (Django + Gunicorn)
  • Le conteneur frontend (Vue.js compilé + Nginx)

6.2 — Appliquer les migrations de base de données

docker compose exec backend python manage.py migrate

6.3 — Initialiser les données (admin + exemples)

docker compose exec backend python manage.py shell < init_data.py

6.4 — Vérifier que tout fonctionne

docker compose ps

Tous les services doivent afficher le statut Up. Vous pouvez également consulter les logs :

docker compose logs -f

6.5 — Tester l'accès

Ouvrez un navigateur et rendez-vous sur :

ServiceURL
Site publichttp://ADRESSE_IP_DU_VPS
APIhttp://ADRESSE_IP_DU_VPS:8000/api/
Administrationhttp://ADRESSE_IP_DU_VPS/admin/
Identifiants par défaut : admin / admin123
Changez le mot de passe immédiatement après la première connexion.

7 Configurer un nom de domaine

7.1 — Faire pointer le domaine vers votre VPS

Dans l'interface de votre registrar (OVH, Namecheap, Gandi…), créez un enregistrement DNS de type A :

TypeNomValeurTTL
A@ADRESSE_IP_DU_VPS3600
AwwwADRESSE_IP_DU_VPS3600
La propagation DNS peut prendre de quelques minutes à 24 heures. Vérifiez avec :
nslookup votre-domaine.com

7.2 — Mettre à jour les variables d'environnement

Modifiez votre fichier .env avec le vrai domaine, puis relancez :

nano /opt/socopro/.env
# Mettre à jour DJANGO_ALLOWED_HOSTS et VITE_API_URL

docker compose up --build -d

8 Certificat SSL / HTTPS avec Let's Encrypt

Pour activer HTTPS gratuitement, installez Certbot et configurez Nginx :

8.1 — Installer Certbot

sudo apt install -y certbot python3-certbot-nginx

8.2 — Obtenir le certificat

Arrêtez d'abord le conteneur frontend pour libérer le port 80 : 
docker compose stop frontend

sudo certbot --nginx -d votre-domaine.com -d www.votre-domaine.com

Certbot va :

  • Vérifier la propriété du domaine
  • Générer les certificats SSL dans /etc/letsencrypt/
  • Modifier automatiquement la configuration Nginx

8.3 — Copier les certificats dans le projet

sudo cp /etc/letsencrypt/live/votre-domaine.com/fullchain.pem /opt/socopro/frontend/ssl/
sudo cp /etc/letsencrypt/live/votre-domaine.com/privkey.pem /opt/socopro/frontend/ssl/

8.4 — Mettre à jour nginx.conf pour HTTPS

Modifiez frontend/nginx.conf pour ajouter un bloc HTTPS avec les certificats, puis relancez :

docker compose up --build -d

8.5 — Renouvellement automatique

Let's Encrypt expire tous les 90 jours. Ajoutez une tâche cron pour le renouvellement automatique :

sudo crontab -e
# Ajouter la ligne suivante :
0 3 * * * certbot renew --quiet && docker compose -f /opt/socopro/docker-compose.yml restart frontend

9 Démarrage automatique au reboot du serveur

Pour que l'application redémarre automatiquement si le serveur redémarre :

9.1 — Créer un service systemd

sudo nano /etc/systemd/system/socopro.service

Collez le contenu suivant :

[Unit]
Description=SOCOPROCAAP Site Vitrine
Requires=docker.service
After=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/opt/socopro
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down
TimeoutStartSec=0

[Install]
WantedBy=multi-user.target

9.2 — Activer le service

sudo systemctl daemon-reload
sudo systemctl enable socopro.service
sudo systemctl start socopro.service

9.3 — Vérifier le statut

sudo systemctl status socopro.service
